### 5.1 DummyDataItemReader作成

- [ ] `ItemReader<String>` インターフェース実装
- [ ] `@Component` アノテーション付与
- [ ] フィールド定義
  - `@Value("${batch.performance.data-count}") int dataCount`
  - `int currentCount = 0`
- [ ] `read()` メソッド実装
  - `currentCount >= dataCount` なら `null` を返す（終了）
  - カウントアップ
  - ダミーデータ生成（連番付き文字列）
  - 進捗ログ出力（10万件ごと）

**実装ポイント**:
```java
@Override
public String read() {
    if (currentCount >= dataCount) {
        return null;  // 読み込み終了
    }
    currentCount++;
    String data = String.format("ChunkData-%010d", currentCount);
    
    // 進捗表示
    if (currentCount % 100000 == 0) {
        System.out.println("読み込み: " + currentCount + " 件");
    }
    
    return data;
}
```

### 5.2 DummyDataBatchItemWriter作成

**ファイル**: [`demo/src/main/java/com/example/demo/presentation/writer/DummyDataBatchItemWriter.java`](../demo/src/main/java/com/example/demo/presentation/writer/DummyDataBatchItemWriter.java)

- [ ] パッケージ作成: `presentation/writer/`
- [ ] クラス作成: `DummyDataBatchItemWriter`
- [ ] `ItemWriter<String>` インターフェース実装
- [ ] `@Component` アノテーション付与
- [ ] フィールド定義
  - `JdbcTemplate jdbcTemplate` (DI)
  - `int totalWriteCount = 0`
- [ ] `write()` メソッド実装
  - SQL文定義: `INSERT INTO dummy_record ...`
  - `jdbcTemplate.batchUpdate()` でバルクInsert
  - 処理件数カウント
  - 進捗ログ出力（チャンクごと）

**実装ポイント**:
```java
@Override
public void write(Chunk<? extends String> chunk) throws Exception {
    List<? extends String> items = chunk.getItems();
    
    String sql = "INSERT INTO dummy_record (data_value, created_at) VALUES (?, ?)";
    
    // バルクInsert（高速）
    jdbcTemplate.batchUpdate(sql, items, items.size(), (ps, dataValue) -> {
        ps.setString(1, dataValue);
        ps.setObject(2, LocalDateTime.now());
    });
    
    totalWriteCount += items.size();
    System.out.println("書き込み: " + totalWriteCount + " 件");
}
```

**確認方法**:
```bash
# 実行ログに書き込み進捗が表示されること
# "[Writer進捗] 10,000 件書き込み完了"
```

```java
private String getJobMode(String[] args) {
    for (String arg : args) {
        if (arg.startsWith("--job=")) {
            return arg.substring(6).toLowerCase();
        }
    }
    return "both";  // デフォルト
}

switch (jobMode) {
    case "tasklet":
        jobLauncher.run(taskletPerformanceJob, jobParameters);
        break;
    case "chunk":
        jobLauncher.run(chunkPerformanceJob, jobParameters);
        break;
    case "both":
        // 両方実行
        break;
}
```

## 🚨 注意事項

### メモリ設定

100万件のデータ処理では、JVMヒープサイズの調整が必要な場合があります。

```bash
# メモリ不足エラーが発生した場合
export MAVEN_OPTS="-Xms512m -Xmx2g"
mvn spring-boot:run
```

### データベースクリーンアップ

テスト間でデータをクリアする場合：

```sql
-- H2コンソールで実行
TRUNCATE TABLE dummy_record;
```

または、アプリケーション再起動（インメモリDBのため自動クリア）

### 実行時間の変動

実行時間は以下の要因で変動します：
- CPU負荷
- メモリ使用状況
- ディスクI/O
- JVMウォームアップ

正確な比較のため、複数回実行して平均を取ることを推奨します。
